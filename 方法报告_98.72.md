# 红色字符识别 — 方法报告（平台 98.72，最终版）

> 机器学习大作业问题三。本报告记录达到 Kaggle 平台 **98.72%** exact-match 的完整方法、各组件设计、消融结论与复现步骤。最后更新：2026-06-16（**已定稿**）。
> 最终提交：`outputs/submission_reranker_gfrl.csv`（平台 0.9872）。交接细节见 `HANDOFF.md`。

---

## 1. 任务与数据

- **任务**：给定 200×60 的 RGB 验证码图，每张恰好 5 个字符（字符集 `0-9A-Z`，36 类），每个字符位有"红/非红"属性。输出**按从左到右顺序排列的全部红色字符**组成的字符串（长度 0~5）。
- **评价指标**：整串 **exact-match** 准确率（5 个红字全对才算对）。
- **数据**：50,000 训练图 + `labels.csv`（`filename,color,all_label`，如 `00000.png,ruuur,DPVKD`，r=红/u=非红）+ 5,000 测试图。
- **划分**：固定随机种子做 95/5 划分 → 47,500 训练 / 2,500 验证；验证集划分由独立固定种子决定，**改训练种子不影响验证集**，保证集成/消融可比。
- **关键观察**：exact ≈ (每位 char&color 联合正确率)^5 → **字符识别是瓶颈**；颜色判定相对容易。

---

## 2. 整体架构（两级：主模型集成 + 高清局部重排）

```
                输入图 200×60 RGB
                       │
        ┌──────────────┴───────────────┐
        ▼                              ▼
  主模型集成(12 个 CNN)           glyph 局部模型(2 个)
  + 水平 TTA(0/-4/+4px)          对每个字符位裁 60×64 高清局部块
        │                          只在红色字符上、用"红度"通道训练
   每位 char 概率[5,36]            每位 char 概率[5,36]
   每位 color 概率[5,2]                 │
        └──────────────┬───────────────┘
                       ▼
            选择性重排(selective rerank)
   仅当 主模型 Top-2 margin ≤ 0.40 且 glyph 候选 margin ≥ 0.05 时，
   用 glyph 在主模型 Top-3 候选内的选择覆盖主模型的字符预测。
   颜色由主模型决定(红判定阈值 τ=0.20)。
                       │
                       ▼
          按位置序拼接红色字符 → submission
```

**核心思想**：主模型负责"全局语境 + 颜色判定"；glyph 模型在高分辨率局部块上专攻"难字细节",**只在主模型没把握时介入**,避免误伤已正确的预测。

## 3. 主模型（Primary，12 个 CNN 集成）

### 3.1 骨干 RedCharNetV2Hi（`model.py`）
- 残差块 + Squeeze-Excite 通道注意力 + CoordConv（注入绝对列位置，契合"读第 i 个字符"）。
- **高分辨率（关键改进）**：只做 3 次降采样（60×200 → 7×25），而非常规 4 次（→3×12）。这样每个字符在特征图上保留约 5px 横向细节，而不是 2-3px——直接缓解 O/0、G/C 这类**细节形状**混淆。
- 1×1 卷积压通道后再 Flatten（保留按位列信息，不做全局池化），控制 FC 规模避免过拟合。约 7.2M 参数。
- 输出：`char_logits[B,5,36]` + `color_logits[B,5,2]`。

### 3.2 12 个成员的来源
| 成员 | 数量 | 说明 |
|---|---|---|
| `v2hs1..6` | 6 | v2hi 不同随机种子 |
| `ps1..3` | 3 | **伪标签自训练**学生（教师=6 个 v2hi） |
| `r2ps1..3` | 3 | 第二轮伪标签学生（教师=round1 集成） |

### 3.3 训练细节
- 损失：`mean_i CE(char_i) + CE(color_i)`，字符头轻量 label smoothing(0.05)。
- AdamW，lr 1.5e-3，weight decay 1e-4，2 epoch warmup + 余弦退火，梯度裁剪，AMP，TF32。
- **权重 EMA**（decay 0.999），按验证 exact 存 best。
- **在线保色增强**：小幅仿射(平移/缩放/旋转) + 轻噪。**严禁色相/饱和度/通道交换/灰度化**——红色是监督信号。

### 3.4 伪标签自训练（平台主力增益之一）
用 6-seed v2hi 集成给 5,000 测试图打标签，**保留 5 个位置 char 置信度 ≥0.92 且 color 置信度 ≥0.90 的**（注：6 模型平均 softmax 峰值约 0.97，阈值用 0.99x 会一张都留不下，必须 ~0.92）。约保留 3,950 张并入训练。验证集保持纯净（真标签、不增强），指标诚实。
> 本质是**向测试分布做自适应**，把平台从 0.9794 推到 0.9810。

### 3.5 推理
- **水平 TTA**：对原图与左右各 4px 平移共 3 个版本求 softmax 平均（不做大位移/几何 TTA，否则会错位按位字符头）。
- **红色判定阈值 τ=0.20**：颜色错误系统性单向（漏红 r→u），把红判定阈值从 0.5 下调到 0.20~0.40 是一整段验证集平台，可救回漏红、无副作用。

## 4. glyph 局部高清重排器（字符识别专家）

### 4.1 动机
主模型把整张图压到 7×25，单字符细节仍有限。glyph 模型对每个**名义字符位**裁一个 60×64 的局部块（比整图里每个字符大得多），在高分辨率下专门重认难字。

### 4.2 GlyphNet（`glyph.py`，最终用 `gfrl` 配置）
- **hires**：只池化 1 次，保留 30×32 特征图（竖笔画 serif、弧形开口等细节翻倍）。
- **GAP 头**：全局平均池化 + 小分类头（约 1.9M 参数），避免大 FC 过拟合（flat 大 FC 版实测过拟合、更差）。
- **红度输入模式 `input_mode=red`（关键改进）**：在 RGB 外附加两个通道——红度图 `relu(R−max(G,B))` 与暗度图。红度图让**红字突出、非红字符与非红干扰线被抑制**，等于把"红字被各色线划"简化为"红字只被红线划"。
- **红线增强 `red_line_p=0.5`（关键改进）**：训练时随机叠加合成红线穿过字符（不改标签），专门训练模型在**残留红线**下也能读对——补上"红度隔离只能去非红线"的短板。
- 训练：仅在红色字符位上训练（部署只对红位重排），全字符训练实测略差。
- 最终用 2 个种子 `gfrl1`、`gfrl2`。

### 4.3 选择性重排（`eval_reranker.py` / `predict_reranker.py`）
```
对每个字符位：
  取主模型 Top-3 候选
  primary_margin = 主模型 Top-1 − Top-2 概率
  glyph_margin   = glyph 在候选内 Top-1 − Top-2 概率
  若 primary_margin ≤ 0.40 且 glyph_margin ≥ 0.05：用 glyph 的选择覆盖
  否则保持主模型预测
```
只在"主模型犹豫 + glyph 有把握"时覆盖，最大化收益、最小化误伤。

## 5. 结果与消融

### 5.1 平台进展
| 方法 | 验证集 exact | 平台分 |
|---|---|---|
| v2×4 集成（起点） | 0.9856 | 0.9700 |
| v2hi 高分辨率 ×3 | 0.9884 | 0.9794 |
| 伪标签自训练 ×3 | 0.9900 | 0.9810 |
| + glyph reranker(gf1, rgb) | 0.9928 | 0.9840 |
| + glyph hires+GAP(gfg) | 0.9936 | 0.9856 |
| + glyph 红度模式(gfr) | 0.9940 | 0.9860 |
| **+ glyph 红度 + 红线增强(gfrl)** | **0.9944** | **0.9872** |

### 5.2 验证集残余错误（gfrl，14/2500）
- **全部为字符错误，零颜色错误**（颜色已基本解决）。
- 集中在三个混淆群：**S 群**(S↔6/8/E/C)、**竖笔画群**(I/1/V/Z 互混)、**G↔C**。
- 成因（看图确认）：目标红字偏小/偏暗 + 粗线遮挡；约一半人可辨别、其余接近难度极限。

### 5.3 有效的关键技术（按贡献）
1. 高分辨率骨干 v2hi（少降采样保字符细节）
2. 多 seed 集成 + 伪标签自训练（向测试分布自适应）
3. glyph 高清局部选择性重排
4. glyph 红度输入（隔离非红线）+ 红线增强（抗残留红线）
5. 红判定阈值下调（救漏红）

### 5.4 试过但无效（已排除，避免重复）
- ImageNet 预训练 ResNet18 骨干（域差距 + 丢 CoordConv 位置信息，更差）
- 强增强（仿射加大+透视+高斯模糊；模糊抹掉细节，更差）
- 第二轮伪标签迭代不复利
- 去线 U-Net（合成域差距导致伪影伤字符/颜色：直接用更差，6 通道融合持平）
- glyph 全字符训练、flat 大 FC hires（过拟合）、宽裁切（邻字干扰）
- 红线增强用到主模型（被强集成稀释，无增益）
- **I/1 竖笔画群过采样(gfb)**：验证集涨 1 样本(0.9944→0.9948)，但**平台反降 0.9872→0.9864**——典型 val 噪声不转化为平台收益。
- 训练集 vs 验证集差距经测为不可约字形歧义，非过拟合（强增强使两者同降证实）

> **测量地板教训**：在 ≥0.994 这个精度，2,500 验证集的 ±1~2 个样本是噪声，**无法预测平台**（gfb 即为反例）。后续任何调参/改进必须用 K 折 OOF（全量 50,000）或更大留出集判断，**平台是唯一裁判，且已进入平台期**。要冲更高最可能需验证码生成器/确切字体做零域差距合成训练（推测同学 99.9% 来源）。

## 6. 复现步骤

环境：conda 环境 `red_char`（`/home/duxuanzheng/.conda/envs/red_char/bin/python`，torch 2.6.0+cu124）。

```bash
cd red_char
PY=/home/duxuanzheng/.conda/envs/red_char/bin/python
CK=outputs/checkpoints

# 1) 6 个 v2hi 主模型（不同种子）
for s in 1 2 3 4 5 6; do $PY train.py --model v2hi --epochs 40 --seed $s --tag _v2hs$s; done

# 2) 伪标签自训练（round1 教师=6 v2hi；round2 教师=round1 集成）
TEACH="$CK/best_v2hs1.pt ... $CK/best_v2hs6.pt"
for s in 11 12 13; do $PY train_pseudo.py --teacher $TEACH --model v2hi --epochs 40 \
   --seed $s --tag _ps$((s-10)) --char-th 0.92 --color-th 0.90; done
# round2 同理，教师换成 best_ps1/2/3.pt，tag _r2ps*

# 3) glyph 局部模型（红度 + hires + GAP + 红线增强），2 个种子
for s in 1 2; do $PY train_glyph.py --hires --head-mode gap --input-mode red \
   --red-line-aug 0.5 --epochs 30 --seed $s --tag _gfrl$s; done

# 4) 验证集评测（端到端 selective rerank）
PRIM="$CK/best_ps1.pt $CK/best_ps2.pt $CK/best_ps3.pt $CK/best_r2ps1.pt $CK/best_r2ps2.pt $CK/best_r2ps3.pt \
      $CK/best_v2hs1.pt $CK/best_v2hs2.pt $CK/best_v2hs3.pt $CK/best_v2hs4.pt $CK/best_v2hs5.pt $CK/best_v2hs6.pt"
$PY eval_reranker.py --x-tta --top-k 3 --checkpoints $PRIM \
   --glyph-checkpoints $CK/best_gfrl1.pt $CK/best_gfrl2.pt

# 5) 生成提交（98.72 版）
$PY predict_reranker.py --x-tta --selective --top-k 3 \
   --primary-margin-max 0.40 --glyph-margin-min 0.05 --red-threshold 0.20 \
   --checkpoints $PRIM --glyph-checkpoints $CK/best_gfrl1.pt $CK/best_gfrl2.pt \
   --output outputs/submission_reranker_gfrl.csv
```

**最终提交文件**：`outputs/submission_reranker_gfrl.csv`（平台 98.72）。

# 红色字符识别 — 完整方法报告（Kaggle 平台 98.98%，合规最终版）

> 本报告**完整自洽**:不依赖任何"历史版本"即可理解与复现。
> 最终提交:`red_char/outputs/submission_compliant.csv`(平台 exact-match **0.9898**)。
> **合规性**:最终系统**不使用预训练模型、不使用任何外部/泄露数据、不对测试集做人工标注、最终提交不含测试集伪标签**(伪标签仅作探索,见 §8,未纳入最终)。

---

## 1. 任务与数据

- **输入**:200×60 RGB 验证码,每张恰好 5 个字符(字符集 `0-9A-Z`,36 类),每位有"红/非红"属性;图上有贯穿全图的彩色干扰线、浅噪点背景、字符位置/大小/旋转抖动、橙色等非红干扰。
- **输出**:把红色字符按从左到右顺序拼成字符串(长 0~5)。
- **指标**:整串 **exact-match** 准确率。
- **数据**:50,000 训练图 + `train/labels.csv`(`filename,color,all_label`,如 `00000.png,ruuur,DPVKD`)+ 5,000 测试图。
- **划分**:固定种子 95/5 → 47,500 训练 / 2,500 验证;验证集由独立固定种子决定,任何训练种子都不改变它(保证集成与消融可比)。
- 关键:exact ≈ (每位 char&color 联合正确率)^5 → **字符识别是瓶颈**,**干扰线遮挡**是最难点。

---

## 2. 总体架构(两级:主模型集成 + 高分辨率局部重排)

```
                       输入 200×60 RGB
            ┌────────────────────┴───────────────────┐
            ▼                                        ▼
   主模型集成(15 个 CNN,全部从零训练)        glyph 局部模型(3 个)
   + 水平 TTA(原图 / 左右各 4px)            对每个名义字符位裁 60×64 局部块
   每位 char[5,36] / color[5,2] 概率          每位 char[5,36] 概率
            └────────────────────┬───────────────────┘
                                 ▼
                   选择性重排(selective rerank)
   每位取主模型 Top-3 候选;当主模型犹豫(Top1−Top2≤0.50)且
   glyph 在候选间有把握(margin≥0.05)时,用 glyph 选择覆盖;否则保留主模型。
   颜色由主模型决定(P(red)≥0.20 判红)。
                                 ▼
                  按位置序拼接红色字符 → submission
```

**动机**:主模型看整图,管语境与颜色;glyph 在高分辨率局部块上专攻被干扰线破坏的难字,**只在主模型没把握时介入**,补强难字又不误伤已对的位。

## 3. 主模型(Primary,15 个 CNN 概率平均;全部从零训练)

骨干为 **RedCharNetV2Hi**;15 个成员 = **9 个标准 v2hi(`v2hs`)+ 6 个重干扰长训 v2hi(`phl`)**。两者结构相同、训练配方不同。

### 3.1 骨干 RedCharNetV2Hi（`model.py`）
输入 `3×60×200`,像素 /255 归一化(从零训练,无需 ImageNet 统计量)。
- **首层 CoordConv**:拼接归一化 x/y 坐标通道再 1×1 卷积,注入绝对列位置(契合按位读字)。
- **4 个残差 + Squeeze-Excite 块**,通道 (48,96,192,384);每块 = 两层 3×3 Conv-BN-ReLU + SE 通道注意力 + 残差。**只前 3 块做 2×2 池化**(第 4 块不池化)→ 60×200 仅降到 **7×25**(保高分辨率,每字符约 5px 细节,缓解 O/0、G/C、S/5 细节混淆)。
- **Neck**:1×1 卷积 384→48 压通道 → Flatten → Linear(48·7·25→512)→BN→ReLU→Dropout(0.3)。用 Flatten(非全局池化)保留按位列信息。
- **双头**:`char_head`(512→5×36)、`color_head`(512→5×2)。约 7.2M 参数。

### 3.2 标准 v2hi 成员(`v2hs1..9`,9 个)
- 损失:`mean_i CE(char_i) + mean_i CE(color_i)`;字符头 label smoothing 0.05(颜色头不平滑)。
- AdamW,LR 1.5e-3,weight decay 1e-4,2 epoch warmup + 余弦退火(η_min 1e-5),梯度裁剪 5.0,AMP,TF32。
- **权重 EMA(0.999)**,按验证 exact 存 best。批 256,**40 epoch**,9 个不同随机种子。
- **在线保色增强**:小幅仿射(平移 ±6%、缩放 0.90~1.10、旋转 ±4°)+ 高斯噪声 std 0.02。**严禁色相/饱和度/通道交换/灰度化**(红色是颜色头监督信号)。

### 3.3 重干扰长训 v2hi 成员(`phl1..6`,6 个;更强)
与 3.2 完全相同的骨干与优化,**仅两处加强**(本方案的关键配方之一):
1. **红线遮挡增强(p=0.6)**:训练时随机在整图上叠加 1~5 条合成红线(颜色取自红色域、宽度 1~3px、随机角度贯穿),模拟"红色干扰线穿过红字"。
2. **长训 100 epoch**(而非 40)。
效果:单模型字符准确率从 **0.990 → 0.994**(干净 2,500 验证集上量),6 个不同种子。
> 注:重增强**必须配长训**才生效——同样增强只训 40ep 无提升;补足 epoch 后才涨。

### 3.4 推理:水平 TTA + 红色判定阈值
- **水平 TTA**:原图、左移 4px、右移 4px 三版求 softmax 平均(只做水平小位移;大位移/几何 TTA 会错位按位字符头,不用)。
- **红色阈值 τ=0.20**:颜色错误系统性单向(漏红),判红阈值从 0.5 降到 0.20,验证集上为一整段平台,救漏红无副作用。

## 4. glyph 局部高分辨率重排器（`ghl`，本方案制胜组件）

独立的字符识别专家,在每个字符位的高分辨率局部块上重判,并选择性纠正主模型。

### 4.1 局部裁切（`glyph.py: extract_glyph_crops`）
以 5 个名义字符位中心为基准,各裁 **60×64** 的重叠局部块(边缘位用白色 padding 等大;裁切宽于单字符,容忍位置抖动)。一个字符在 60×64 局部块里远大于整图中,高分辨率细节得以保留。

### 4.2 输入表示 `red2`（强度鲁棒红度,5 通道）
喂网络的是 5 通道 `[R,G,B,redness,darkness]`:
- `redness = relu(R − max(G,B))`,再**按每个局部块自身最大值归一化** `redness ← redness/(max+1e-4)`。
  - 普通绝对红度对**浅红/不均匀红**笔画(R 仅略大于 G/B)值极小、会被当背景丢掉(浅红 V 的右笔消失→被读成 I)。**按块归一化**让浅红笔画相对该块最强红被提到可见范围,保住完整字形。
- `darkness = 1 − mean(RGB)`:任何非背景墨迹(含浅红)都有暗度,补充形状信号。
- 红度表示同时**抑制非红干扰线**(非红线 redness≈0),突出红字。

### 4.3 网络（`GlyphNet`，hires + GAP）
- 主干:4 个残差+SE 块,通道 (48,96,192,256),**只 1 次池化**→ 60×64 降到 **30×32**(比主模型更高的局部分辨率,保竖笔画 serif、弧形开口)。
- **GAP + 小分类头**:AdaptiveAvgPool2d→Flatten→Linear(256→512)→BN→ReLU→Dropout(0.2)→Linear(512→36)。约 1.9M 参数。(高分辨率特征直接 Flatten 会得超大 FC、严重过拟合;GAP 头保细节又不过拟合。)
- 仅在**红色字符**上训练。

### 4.4 训练:重干扰增强 + 长训（关键）
在 47,500 训练图的红字局部块(约 11.9 万裁切)上训,**100 epoch**,AdamW+EMA(0.99)+余弦退火,3 个种子。增强**全部保色**:
1. **几何**:平移 ±8%、缩放 0.94~1.06、旋转 ±5°、噪声 std 0.015。
2. **红线遮挡(p=0.7)**:随机 **1~5 条合成红线**(红色域、宽 1~3px、贯穿、随机角度)——直接练"红线穿红字"这一最难情形(红度表示去不掉红线,只能靠此增强)。
3. **浅红/不均匀红渐变(p=0.4)**:沿随机方向加线性渐变,把一侧墨迹按比例淡化向白背景,模拟"左深右浅"浅红笔画(否则浅笔被丢、V→I)。
4. **遮挡块 cutout(p=0.3)**:随机擦除 1~2 个小矩形(每边≤35%),练"看不全也能认"。

### 4.5 效果（held-out 6,231 验证红字）
| glyph | 整体红字准确率 | V | I | S |
|---|---|---|---|---|
| 无重干扰增强/短训(对照) | 0.99695 | 0.9953 | 0.9760 | 0.9765 |
| **ghl(red2 + 重增强 + 100ep)** | **0.99872** | **1.0000** | **0.9940** | **0.9882** |

红字错误 **19→8**,V/I/S 等被干扰线破坏的难字显著改善。

## 5. 选择性重排（`predict_reranker.py`）

```
候选 = 主模型该位 Top-3
primary_margin = 主 Top1 − Top2
glyph_margin   = glyph 在候选内 Top1 − Top2
若 primary_margin ≤ 0.50 且 glyph_margin ≥ 0.05:  用 glyph 候选最高者(纠错)
否则:                                          保留主模型 Top1
颜色:P(red) ≥ 0.20 → 该位计红;按序拼接所有红位字符
```
即"主模型犹豫 + glyph 有把握"才覆盖。阈值 `primary_margin_max=0.50, glyph_margin_min=0.05, red_threshold=0.20, top_k=3`。

### 关键消融(有效 vs 无效)
- **有效**:高分辨率主干(v2hi);多 seed 概率平均集成;glyph 高分辨率局部选择性重排;**red2 强度鲁棒红度**;**重红线遮挡增强**;**浅红渐变增强**;**100ep 长训(关键,同样增强 40ep 无效)**;红判定阈值 0.20。
- **无效/已排除(勿重试)**:ImageNet 预训练 ResNet18/大模型(域差距+过拟合);强增强含高斯模糊(抹掉细节);单独在合成数据上训的去线 U-Net(域差距→伪影);端到端去噪前端(被主干隐式去噪覆盖);glyph 宽裁切(邻字干扰);I/1 过采样(val 涨平台跌);把红线增强用到主模型后**进一步加 seed/轮数**(饱和:18 主 vs 15 主仅差 1 张预测);各种推理阈值/TTA 的进一步微调(噪声级,过高 primary_margin 反而掉分)。

## 7. 复现步骤

环境:conda 环境 `red_char`(`/home/duxuanzheng/.conda/envs/red_char/bin/python`,torch 2.6.0+cu124,A100)。多卡每进程限 `OMP_NUM_THREADS=6` 等,`setsid ... </dev/null &` 后台。

```bash
cd red_char
PY=/home/duxuanzheng/.conda/envs/red_char/bin/python
CK=outputs/checkpoints

# 1) 9 个标准 v2hi(40 epoch,标准保色增强)
for s in 1 2 3 4 5 6 7 8 9; do $PY train.py --model v2hi --epochs 40 --seed $s --tag _v2hs$s; done

# 2) 6 个重干扰长训 v2hi(phl:红线增强 0.6 + 100 epoch)
for s in 1 2 3 4 5 6; do $PY train.py --model v2hi --red-line-aug 0.6 --epochs 100 --seed $s --tag _phl$s; done

# 3) glyph 制胜配方 ghl(red2 + 重红线/浅红/遮挡 + 100 epoch),3 个种子
for s in 1 2 3; do $PY train_glyph.py --hires --head-mode gap --input-mode red2 \
   --red-line-aug 0.7 --faint-aug 0.4 --cutout 0.3 --epochs 100 --seed $s --tag _ghl$s; done

# 4) 生成最终合规提交(平台 98.98)
PRIM="$CK/best_v2hs1.pt $CK/best_v2hs2.pt $CK/best_v2hs3.pt $CK/best_v2hs4.pt $CK/best_v2hs5.pt $CK/best_v2hs6.pt \
      $CK/best_v2hs7.pt $CK/best_v2hs8.pt $CK/best_v2hs9.pt \
      $CK/best_phl1.pt $CK/best_phl2.pt $CK/best_phl3.pt $CK/best_phl4.pt $CK/best_phl5.pt $CK/best_phl6.pt"
$PY predict_reranker.py --x-tta --selective --top-k 3 \
   --primary-margin-max 0.50 --glyph-margin-min 0.05 --red-threshold 0.20 \
   --checkpoints $PRIM --glyph-checkpoints $CK/best_ghl1.pt $CK/best_ghl2.pt $CK/best_ghl3.pt \
   --output outputs/submission_compliant.csv
```

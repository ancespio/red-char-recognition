# 红色字符识别 — 完整方法报告（Kaggle 平台 98.96%）

> 本报告**完整自洽**:不依赖任何"历史版本"即可理解与复现。描述的是最终提交系统的全部组件、超参与训练流程。
> 最终提交文件:`red_char/outputs/submission_ghl.csv`(平台 exact-match **0.9896**)。

---

## 1. 任务与数据

- **输入**:200×60 的 RGB 验证码图,每张恰好 5 个字符(字符集 `0-9A-Z`,共 36 类),每个字符位有"红/非红"属性。图上有**贯穿全图的彩色干扰线**、浅色噪点背景、字符位置/大小/旋转抖动,并存在橙色等非红干扰色。
- **输出**:把图中**红色**字符按从左到右顺序拼成字符串(长度 0~5)。
- **评价指标**:整串 **exact-match** 准确率(5 个红字全对才算一张对)。
- **数据**:50,000 训练图 + `train/labels.csv`(每行 `filename,color,all_label`,如 `00000.png,ruuur,DPVKD`;`color` 用 r/u 标 5 位红/非红,`all_label` 是 5 个字符)+ 5,000 测试图 + `submission_sample.csv`。
- **划分**:固定种子 95/5 → 47,500 训练 / 2,500 验证;验证集划分由独立固定种子(`config.SEED=42`)确定,任何训练随机种子都**不改变验证集**,保证集成与消融可比。
- 关键性质:exact ≈ (每位 char&color 联合正确率)^5 → **字符识别是瓶颈**(颜色判定相对易解),且**干扰线遮挡**是最难点。

---

## 2. 总体架构(两级:主模型集成 + 高分辨率局部重排)

```
                       输入 200×60 RGB
            ┌────────────────────┴───────────────────┐
            ▼                                        ▼
   主模型集成(15 个 CNN)                     glyph 局部模型(3 个)
   + 水平 TTA(原图 / 左右各 4px)            对每个名义字符位裁 60×64 局部块
            │                                  红度表示 + 重干扰增强 + 长训
   每位 char 概率[5,36] / color[5,2]        每位 char 概率[5,36]
            └────────────────────┬───────────────────┘
                                 ▼
                   选择性重排(selective rerank)
   每位取主模型 Top-3 候选;当"主模型犹豫(Top1−Top2≤0.50)"且
   "glyph 在候选间有把握(margin≥0.05)"时,用 glyph 的选择覆盖主模型字符;
   否则保留主模型。颜色由主模型决定(P(red)≥0.20 判为红)。
                                 ▼
                  按位置序拼接红色字符 → submission
```

**设计动机**:主模型看整图,负责语境关系与颜色判定;glyph 模型在高分辨率局部块上专攻被干扰线破坏的难字,且**只在主模型没把握时介入**,既补强难字、又不误伤主模型已对的位置。

## 3. 主模型(Primary,15 个 CNN 的概率平均集成)

### 3.1 骨干 RedCharNetV2Hi(`model.py`)
单张图 `3×60×200`,像素归一化到 [0,1](从零训练,不用 ImageNet 统计量)。

- **首层 CoordConv**:在输入上拼接归一化的 x/y 坐标通道再做 1×1 卷积,**注入绝对列位置**(契合"读第 i 个字符"的按位预测)。
- **主干:4 个残差 + Squeeze-Excite 块**,通道 (48,96,192,384)。每块 = 两层 3×3 Conv-BN-ReLU + SE 通道注意力 + 残差跳连。**只在前 3 块做 2×2 最大池化**(第 4 块不池化)→ 60×200 仅降到 **7×25**(而非常规 4 次池化的 3×12)。**保留更高的空间分辨率**,让每个字符在特征图上保留约 5px 宽度的细节,缓解 O/0、G/C、S/5 这类细节形状混淆。
- **Neck**:1×1 卷积把 384 通道压到 48(控制随后全连接规模、防过拟合)→ Flatten → Linear(48·7·25→512) → BN → ReLU → Dropout(0.3)。**用 Flatten 而非全局池化**,以保留按位的列位置信息。
- **双头**:`char_head` Linear(512→5×36)、`color_head` Linear(512→5×2)。输出 `char_logits[B,5,36]` 与 `color_logits[B,5,2]`。约 7.2M 参数。

### 3.2 损失与训练超参
- 损失:`mean_i CE(char_i) + mean_i CE(color_i)`(颜色权重 1.0);**字符头**加轻量 label smoothing **0.05**(颜色头不平滑)。
- 优化:AdamW,峰值 LR **1.5e-3**,weight decay **1e-4**;**2 epoch 线性 warmup + 余弦退火**(η_min 1e-5);梯度裁剪 max-norm 5.0;混合精度 AMP;A100 上开 TF32。
- **权重 EMA**(decay 0.999):用 EMA 权重做验证与导出,按验证 exact-match 存 best。
- **批大小 256,训练 40 epoch。**
- **在线保色增强(仅训练集)**:每 epoch 实时生成。包含小幅仿射(平移 ±6%、缩放 0.90~1.10、旋转 ±4°)+ 轻微高斯噪声(std 0.02)。**严禁色相/饱和度/通道交换/灰度化**——红色是颜色头的监督信号,任何改变颜色的增强都会破坏它。

### 3.3 集成的 15 个成员
| 成员 | 数量 | 训练方式 |
|---|---|---|
| `v2hs1..9` | 9 | RedCharNetV2Hi,9 个不同随机种子(初始化/打乱/增强不同,划分不变) |
| `ps1..3` | 3 | 伪标签自训练学生(见 3.4),教师 = 6 个 v2hi |
| `r2ps1..3` | 3 | 第二轮伪标签学生,教师 = ps 集成 |

推理时对 15 个模型的 **softmax 概率求平均**(平均概率而非 logit,是这类集成稳健提分的标准做法)。

### 3.4 伪标签自训练(`train_pseudo.py`)
1. 用已训好的 v2hi 集成(教师)对 **5,000 测试图**预测每位 char/color 概率;
2. **保留高置信图**:某图被采纳当且仅当其 5 个位置的最小 char 置信度 ≥ **0.92** 且最小 color 置信度 ≥ **0.90**(注:多模型平均后 softmax 峰值约 0.97,阈值若设 0.99x 会一张都留不下,必须用 ~0.92)。约保留 3,900~4,000 张;
3. 把这些测试图按"预测即标签"并入训练集(47,500 真标签 + ~3,950 伪标签),按 3.2 的配方重训 v2hi 学生;**验证集始终只用真标签、不参与伪标签**,保证指标诚实。

> 作用:把测试集图像(仅用图、不用真标签,属合规的 transductive 自适应)纳入训练,使模型向测试分布对齐。第二轮以更强的 ps 集成当教师再做一次。

### 3.5 推理:水平 TTA + 红色判定阈值
- **水平 TTA**:对原图、左移 4px、右移 4px 三个版本求 softmax 平均(只做水平小位移;不做大位移/缩放/旋转 TTA,因为它们会错位按位字符头)。
- **红色判定阈值 τ=0.20**:颜色错误系统性单向(把红判成非红=漏红),把判红阈值从默认 0.5 降到 **0.20**(P(red)≥0.20 即判红)在验证集上是一整段平台,可救回漏红、无副作用。

## 4. glyph 局部高分辨率重排器(`ghl`,本方案的制胜组件)

这是把平台从 ~98.7% 推到 **98.96%** 的关键。它是一个**独立的字符识别专家**,在每个字符位的高分辨率局部裁切上重新判字,并选择性地纠正主模型。

### 4.1 局部裁切(`glyph.py: extract_glyph_crops`)
对每张 200×60 图,以 5 个名义字符位中心为基准,各裁一个 **60×64** 的重叠局部块(左右用白色 padding,使边缘位与中间位等大;裁切宽于单字符,容忍位置抖动不切到目标字)。一个字符在 60×64 局部块里比在整图里大得多 → 高分辨率细节得以保留。

### 4.2 输入表示:`red2`(强度鲁棒的红度,5 通道)
喂给 glyph 网络的不是原始 RGB,而是 5 通道 `[R, G, B, redness, darkness]`:
- `redness = relu(R − max(G,B))`,再**按每个局部块自身的最大值归一化**:`redness ← redness / (max(redness)+1e-4)`。
  - 普通绝对红度对**浅红/不均匀红**笔画(R 仅略大于 G/B)值极小、会被当背景丢掉(于是浅红的 V 右笔消失→被读成 I)。**按块归一化**让浅红笔画相对该块最强红被提到可见范围,保住完整字形。
- `darkness = 1 − mean(RGB)`:任何非背景墨迹(含浅红)都有暗度,作为形状的补充信号。
- 红度表示同时**抑制非红的干扰线**(非红线 redness≈0),只突出红字。

### 4.3 网络结构(`GlyphNet`,hires + GAP 头)
- 主干:4 个残差+SE 块,通道 (48,96,192,256),**只做 1 次池化**(其余不池化)→ 60×64 局部块仅降到 **30×32** 特征图(比主模型更高的局部分辨率,保住竖笔画 serif、弧形开口等细节)。
- **头部:全局平均池化(GAP) + 小分类头** Linear(256→512)→BN→ReLU→Dropout(0.2)→Linear(512→36)。约 1.9M 参数。
  - 注:此处**用 GAP 而非大 Flatten 全连接**——高分辨率特征图直接 Flatten 会得到极大 FC、严重过拟合;GAP 头保住高分辨率卷积细节又不过拟合。
- 仅在**红色字符**上训练(部署时只对红位重排)。

### 4.4 训练:重干扰增强 + 长训(关键)
glyph 在 47,500 训练图的红字局部块上训练(约 11.9 万个红字裁切),**100 epoch**(长训是关键,见消融),AdamW + EMA(decay 0.99)+ 余弦退火,3 个随机种子。增强**全部保色**,逐项如下(每项按概率独立施加):

1. **几何**:平移 ±8%、缩放 0.94~1.06、旋转 ±5°、高斯噪声 std 0.015(基础)。
2. **红线遮挡增强(red-line aug,p=0.7)**:在局部块上随机画 **1~5 条合成红线**(颜色取自红色域:R∈[140,255]、G/B 较低;宽度 1~3px;贯穿局部块、随机角度)。**直接模拟"红色干扰线穿过红字"这一最难情形**(红度表示能去掉非红线,但红线去不掉,只能靠此增强让模型学会在红线下读对)。
3. **浅红/不均匀红渐变(faint aug,p=0.4)**:沿随机方向加一个线性渐变,把一侧墨迹按比例淡化向白背景,**模拟"左深右浅"的不均匀浅红笔画**(否则浅笔会被丢、V→I)。
4. **遮挡块(cutout,p=0.3)**:随机擦除 1~2 个小矩形(每边 ≤35%,随机填充),模拟字符被局部覆盖,练"看不全也能认"。

### 4.5 长训 + 重增强的效果(held-out 6,231 个验证红字上量)
| glyph | 整体红字准确率 | V | I | S |
|---|---|---|---|---|
| 无重干扰增强/短训(对照) | 0.99695 (6212/6231) | 0.9953 | 0.9760 | 0.9765 |
| **本方案 ghl(red2 + 重增强 + 100ep)** | **0.99872 (6223/6231)** | **1.0000** | **0.9940** | **0.9882** |

红字错误 **19→8**(近乎减半),V/I/S 等被干扰线破坏的难字显著改善。

## 5. 选择性重排(`predict_reranker.py: selective_rerank`)

对每个字符位:
```
候选 = 主模型该位 Top-3 字符
primary_margin = 主模型 Top1 − Top2 概率
glyph_margin   = glyph 在这 3 个候选内的 Top1 − Top2 概率
若 primary_margin ≤ 0.50  且  glyph_margin ≥ 0.05:
        采用 glyph 在候选中最高者(纠正难字)
否则:  保留主模型 Top1
颜色:  P(red) ≥ 0.20 → 该位计为红
最终:  按位置序拼接所有"红"位的字符
```
即"主模型犹豫 + glyph 有把握"才覆盖,最大化收益、最小化误伤。最终阈值 `primary_margin_max=0.50, glyph_margin_min=0.05, red_threshold=0.20, top_k=3`。

### 关键消融(有效 vs 无效,供复核)
- **有效**:高分辨率主干(v2hi);多 seed 概率平均集成;伪标签自训练;glyph 高分辨率局部选择性重排;**red2 强度鲁棒红度**;**重红线遮挡增强**;**浅红渐变增强**;**长训(100ep,这是 ghl 跳分的关键——同样增强只训 40ep 无效)**;红判定阈值 0.20。
- **无效/已排除(勿重试)**:ImageNet 预训练 ResNet18/大模型(域差距+过拟合);强增强含高斯模糊(抹掉细节);单独在合成数据上训的去线 U-Net(域差距→伪影伤识别);端到端去噪前端(被主干隐式去噪覆盖,中性);glyph 宽裁切(邻字干扰);I/1 类过采样(val 涨平台跌);把红线增强用到主模型(被 glyph 重排吸收,端到端无增益);各种推理阈值/TTA 的进一步微调(噪声级,过高的 primary_margin 反而掉分)。

## 7. 复现步骤

环境:conda 环境 `red_char`(`/home/duxuanzheng/.conda/envs/red_char/bin/python`,torch 2.6.0+cu124,A100)。多卡时每进程限 `OMP_NUM_THREADS=6` 等并用 `setsid ... </dev/null &` 后台。

```bash
cd red_char
PY=/home/duxuanzheng/.conda/envs/red_char/bin/python
CK=outputs/checkpoints

# 1) 主模型:9 个 v2hi 不同种子(40 epoch,标准保色增强)
for s in 1 2 3 4 5 6 7 8 9; do $PY train.py --model v2hi --epochs 40 --seed $s --tag _v2hs$s; done

# 2) 伪标签自训练:round1 教师=6 个 v2hi;round2 教师=round1 学生
TEACH="$CK/best_v2hs1.pt ... $CK/best_v2hs6.pt"
for s in 1 2 3; do $PY train_pseudo.py --teacher $TEACH --model v2hi --epochs 40 \
   --seed $((10+s)) --tag _ps$s --char-th 0.92 --color-th 0.90; done
# round2: --teacher 换成 best_ps1/2/3.pt, --tag _r2ps$s

# 3) glyph 制胜配方 ghl:red2 + 重红线/浅红/遮挡增强 + 100 epoch,3 个种子
for s in 1 2 3; do $PY train_glyph.py --hires --head-mode gap --input-mode red2 \
   --red-line-aug 0.7 --faint-aug 0.4 --cutout 0.3 --epochs 100 --seed $s --tag _ghl$s; done

# 4) 生成最终提交(平台 98.96)
PRIM="$CK/best_ps1.pt $CK/best_ps2.pt $CK/best_ps3.pt $CK/best_r2ps1.pt $CK/best_r2ps2.pt $CK/best_r2ps3.pt \
      $CK/best_v2hs1.pt $CK/best_v2hs2.pt $CK/best_v2hs3.pt $CK/best_v2hs4.pt $CK/best_v2hs5.pt $CK/best_v2hs6.pt \
      $CK/best_v2hs7.pt $CK/best_v2hs8.pt $CK/best_v2hs9.pt"
$PY predict_reranker.py --x-tta --selective --top-k 3 \
   --primary-margin-max 0.50 --glyph-margin-min 0.05 --red-threshold 0.20 \
   --checkpoints $PRIM --glyph-checkpoints $CK/best_ghl1.pt $CK/best_ghl2.pt $CK/best_ghl3.pt \
   --output outputs/submission_ghl.csv
```

## 9. 铁律
- 增强**绝不**改变颜色(禁色相/饱和度/通道交换/灰度)——红色是监督信号。
- 写 csv 一律 `dtype=str, keep_default_na=False` + LF 行尾,空预测写裸逗号(防 pandas 把 `NA` 当缺失、防空 label 变 nan)。
- 仅用测试集**图像**做伪标签/TTA(合规);绝不使用测试集真标签。
